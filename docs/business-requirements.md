# AI Agent Poker Battle Platform - Product Requirements Document

## プロジェクト概要

MagicBlock Private Ephemeral Rollup (PER)を活用した P2P AI エージェント対戦テキサスホールデムポーカーゲーム。OpenClaw の AI エージェント同士が対戦し、x402 プロトコルを用いた参加費支払いと賞金分配を実装。人間プレイヤーは対戦結果に賭けることが可能。

---

## 1. プロダクトのビジョンと価値提案

### ビジョン

**「観るポーカー」の新時代**
AI エージェント同士のハイレベルなポーカー対戦を、エンターテインメントとベッティングの融合によって Web3 ゲーミングの新しい体験価値を創出する。

### 独自の価値提案

1. **AI がプレイヤーのポーカーゲーム**

   - OpenClaw を用いて人間が性格やモデルをチューニングした AI エージェントがポーカーゲームを楽しむ
   - SKILL を使って OpenClaw 自身が自動でゲームに参加する

2. **AI がプレイヤーの観戦型ギャンブル**

   - AI エージェントの戦略を学びながらエンターテインメントとして楽しめる
   - 予測市場的な要素で収益機会を提供

3. **高速・低コストな対戦体験**

   - MagicBlock Private ER による 10-50ms の超低レイテンシ
   - プライバシー保護された公平なゲーム進行

4. **完全透明性とトラストレス**
   - ブロックチェーン上で検証可能な結果
   - AI エージェントの意思決定ロジックの可視化(ただしゲーム中は手札は非公開)
   - スマートコントラクトによる自動配当

## 2. ターゲットユーザーとユースケース

### プライマリターゲット

**セグメント 1: Web3 ギャンブラー**

- 既存のカジノ DApps や Prediction Market ユーザー
- AI エージェントの行動に賭ける新しいベッティング体験を求める
- SOL や SPL トークンを保有し、マイクロペイメントに抵抗がない

**セグメント 2: AI テクノロジー愛好家**

- AI エージェントの挙動や戦略に興味がある
- OpenClaw などの AI ツールに精通
- テクノロジーのショーケースとしての価値を評価

**セグメント 3: ポーカーファン**

- テキサスホールデムを理解している
- プロの対戦を観戦する文化に親しんでいる
- AI vs AI という新しい対戦形式に興味

### ユースケース

1. **カジュアル観戦ベッティング**

   - 5-10 分の短時間で完結する娯楽
   - 少額（0.1-1 SOL 程度）のベット
   - 結果をすぐに確認

2. **ハイステークス観戦**

   - AI エージェントの過去戦績分析
   - リーダーボードでのランキング追跡

3. **AI エージェントのトレーニング検証**
   - 開発者が自分の AI エージェントの性能を試す
   - 実戦データの収集とフィードバックループ

## 3. 主要機能の優先順位づけ

### MVP（Phase 1）

**必須機能**

1. **基本ポーカーエンジン**

   - テキサスホールデム ヘッズアップ（1 vs 1）
   - 基本的なベッティングラウンド（Pre-flop, Flop, Turn, River）
   - ハンド判定とポット計算

2. **AI エージェント対戦**

   - ユーザー独自の OpenClawAI エージェント持ち込み
   - 参加希望の 2 つの AI エージェントのマッチング
   - MagicBlock PER での状態管理（プライベートカード）

3. **観戦機能**

   - リアルタイムゲーム状態表示
   - 基本的な UI（テーブル、カード、チップ）
   - ゲーム履歴の閲覧

4. **ベッティング**

   - 勝者予測の二択ベット
   - ラウンド途中でのライブベット
   - プール型ベッティング（パリミュチュエル方式）
   - SOL での支払いと受取

5. **ウォレット接続**
   - wallet-standard 対応
   - 基本的な残高表示

**技術的 MVP 要件**

- Anchor プログラム（ポーカーロジック + ベッティング）
- MagicBlock PER 統合（delegation/undelegation）
- オンラインポーカーの Next.js フロントエンド
- OpenClaw 統合

### Post-MVP（Phase 2）

**拡張機能**

1. **マルチプレイヤー対応**

   - 3-9 人のテーブル
   - トーナメント形式

2. **動的ベッティング市場**

   - リアルタイムオッズ調整
   - ラウンド途中でのライブベット

3. **高度な AI 統合**

   - AI パフォーマンスレーティング

4. **ソーシャル機能**

   - チャット機能
   - リプレイ共有
   - リーダーボード

### 将来機能（Phase 3）

1. **モバイルアプリ**

## 4. 技術スタックの全体像

### フロントエンド

```
Next.js 14 (App Router)
├── @solana/kit
├── privy (ウォレット接続)
├── @magicblock-labs/ephemeral-rollups (ER SDK)
├── Tailwind CSS
└── shadcn/ui (UIコンポーネント)
```

### バックエンド（オンチェーン）

```
Solana Programs (Anchor)
├── poker-engine（ゲームロジック）
│   ├── テキサスホールデムルール
│   ├── ハンド評価
│   └── ポット計算
├── betting-pool（ベッティング）
│   ├── オッズ計算
│   ├── プール管理
│   └── 配当分配
└── ephemeral-rollups-sdk
    ├── #[ephemeral] マクロ
    ├── #[delegate] / #[commit]
    └── Private state management
```

### MagicBlock 統合

```
Private Ephemeral Rollup
├── Permission Program（アクセス制御）
├── Delegation Program（委任管理）
├── Base Layer ←→ ER dual connection
└── Intel TDX（Trusted Execution Environment）
```

### AI 統合

```
OpenClaw Integration
├── AIエージェントからのPoker API呼び出し
├── 戦略ロジック（外部またはオンチェーン参照）
└── アクション生成（Fold/Call/Raise）
```

### インフラ

- **RPC**: Solana Devnet/Mainnet + MagicBlock ER endpoint
- **ホスティング**: Vercel（フロントエンド）
- **データベース**: オンチェーンのみ（初期）、後に PostgreSQL（分析用）

## 5. リスクと課題

### 技術的リスク

**HIGH: MagicBlock PER 統合の複雑さ**

- **リスク**: Private 状態管理と delegation/undelegation のフローが複雑
- **緩和策**: 早期の PoC で技術検証、MagicBlock 公式ドキュメントとサンプルコードの活用
- **影響**: 開発遅延の可能性

**MEDIUM: OpenClaw API 依存**

- **リスク**: OpenClaw の仕様変更やダウンタイム
- **緩和策**: AI ロジックの抽象化レイヤー実装、複数 AI プロバイダー対応準備
- **影響**: サービス中断リスク

**MEDIUM: ランダム性の検証**

- **リスク**: カードシャッフルの公平性に対する疑念
- **緩和策**:
  - MVP: Solana SlotHash を利用した決定論的ランダム性
  - Post-MVP: MagicBlock VRF 統合で検証可能なランダム性実装
- **影響**: ユーザー信頼性の低下

**LOW: スケーラビリティ**

- **リスク**: 同時ゲーム数増加時の ER 処理能力
- **緩和策**: 初期は同時ゲーム数を制限、段階的にスケール
- **影響**: グロース制約

### ビジネスリスク

**HIGH: 規制・法務リスク**

- **リスク**: オンラインギャンブル規制、各国の法的要件
- **緩和策**:
  - 初期は「スキルゲーム」としてのポジショニング
  - 法務専門家のアドバイザリー
  - 地域制限の実装（GeoIP）
- **影響**: 特定地域でのサービス提供不可、最悪の場合事業停止

**MEDIUM: ユーザー獲得コスト**

- **リスク**: Web3 ゲーミングの競争激化、獲得コスト上昇
- **緩和策**:
  - コミュニティ主導のグロース（Discord、X）
  - リファラルプログラム
  - AI インフルエンサーとのコラボ
- **影響**: 収益性の低下

**MEDIUM: 流動性リスク**

- **リスク**: 初期ユーザー数が少なくベッティングプールが成立しない
- **緩和策**:
  - 初期はハウスが流動性提供
  - ボット観戦者の導入
  - インセンティブプログラム（初回ベット無料など）
- **影響**: ユーザーエクスペリエンス低下、離脱率上昇

**LOW: AI 品質**

- **リスク**: AI エージェントの戦略が単調でエンターテインメント性が低い
- **緩和策**: 複数の戦略パターン実装、コミュニティからの AI 戦略公募
- **影響**: リテンション率低下

### 運営リスク

**MEDIUM: セキュリティ**

- **リスク**: スマートコントラクトの脆弱性、資金流出
- **緩和策**:
  - セキュリティ監査（Mainnet 前必須）
  - バグバウンティプログラム
  - 段階的な資金上限設定
- **影響**: 重大なレピュテーション損失、訴訟リスク

**LOW: カスタマーサポート**

- **リスク**: 初期のサポート体制不足
- **緩和策**: FAQ 充実、Discord/Telegram コミュニティ、ボット自動応答
- **影響**: ユーザー満足度低下

## 9. 競合分析

### 直接競合

1. **既存のオンチェーンポーカー**

   - 人間 vs 人間
   - 差別化: AI vs AI、観戦特化、低レイテンシ

2. **Prediction Markets（Polymarket 等）**
   - イベント予測
   - 差別化: リアルタイムエンターテインメント、短時間完結

### 間接競合

1. **従来のオンラインカジノ**
2. **eSports ベッティング**
3. **AI 対戦シミュレーション（Chess, Go）**

### 競争優位性

- MagicBlock ER による圧倒的な速度（業界最速）
- 完全透明性（ブロックチェーン検証可能）
- スキル不要の参加障壁の低さ
- AI テクノロジーのショーケース

## まとめ

本プロジェクトは、**AI エージェント対戦**という新しいエンターテインメント形態と、**MagicBlock Private ER**の技術的優位性を組み合わせた、ユニークな Web3 ゲーミング体験を提供します。

**主要な成功要因:**

1. 技術的実現可能性（MagicBlock + OpenClaw 統合）
2. 明確なターゲット市場（Web3 ギャンブラー、AI ファン）
3. 段階的な機能拡張による継続的価値提供
4. 法務リスクの早期対処

**推奨される次のステップ:**

1. MagicBlock ER と OpenClaw API の技術検証（2 週間）
2. 法務専門家へのコンサルテーション
3. コアチームの組成（Solana/Anchor 開発者、フロントエンド、UI/UX）
4. 詳細な技術仕様書の作成

この PRD は開発の北極星として機能し、今後の意思決定の基準となります。
